## 2026-03-20 - Replace Multiple Stream Operations with Traditional Loop in Critical Render Paths
**Learning:** In frequently executed paths, such as the shopping cart price calculation (`CheckDataRender`, `CartServiceImpl`), chaining multiple stream `.filter(...).mapToInt(...).sum()` operations incurs noticeable object allocation and iteration overhead compared to a single traditional `for` loop that aggregates values. While Stream API provides declarative readability, inside loops processing user shopping cart data, doing repeated passes on identical lists multiplies the overhead.
**Action:** Identify multi-pass stream filtering and summing (especially in inner loops or core rendering algorithms). Replace them with single-pass `for` loops that conditionally aggregate values into local primitives to cut down on redundant iterations and GC pressure.
